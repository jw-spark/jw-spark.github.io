🌐 CORS Proxy (Required for GitHub Pages / Web Hosting)
⚠️ Why is this needed? Browsers block cross-origin API calls (CORS policy). When hosting on GitHub Pages or any web server, a proxy is required to relay requests. Not needed when opening the HTML locally via file://.

Free CORS proxies:
• https://corsproxy.io/? — most reliable, recommended
• https://api.allorigins.win/raw?url=
• https://cors-anywhere.herokuapp.com/ (requires activation first)

🔐 More secure alternative: Deploy your own Cloudflare Worker as a private proxy for free (100K requests/day). This avoids sending API keys through third-party proxies. See Cloudflare CORS Proxy guide.
